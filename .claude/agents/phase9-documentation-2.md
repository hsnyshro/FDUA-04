## 🎯 使命

Phase8のフィードバックを反映し、事業計画書の最終版を完成させます。12要素すべてを統合した完全な事業計画書（10,000字程度）を作成します。

**対応する12要素**: ①～⑫ すべての要素を完成化

---

## 入力データ
output/[企業コード番号]/ 配下のファイル

## 📋 実施内容

### 1. フィードバック反映

- **Phase7の draft_business_plan.md** と **Phase8の validation_report.md** を読み込み
- **改善指摘を分類**: 必須改善、推奨改善、検討事項に分類
- **各改善を実施**: 施策内容、実装順序、リソース、財務計画を修正
- **整合性確認**: 改善後の全体での矛盾がないか確認

### 2. 12要素の完成化

各要素の完成度を最大化：

| # | 要素 | 最終化項目 |
|----|------|-----------|
| ① | ビジョン・目標 | 明確で説得力のあるビジョン記述 |
| ② | ビジネスコンセプト | Who, What, How が完全に統合 |
| ③ | ターゲット市場 | 市場セグメント、規模、成長性が明確 |
| ④ | 市場環境 | PEST分析、業界トレンドが網羅 |
| ⑤ | 競合分析 | 競合優位性が明確、ポジショニング確定 |
| ⑥ | ビジネスモデル | 価値提案、収益モデルが具体化 |
| ⑦ | マーケティング戦略 | 4P施策が実行可能レベルに具体化 |
| ⑧ | 損益計画 | 売上予測、損益計画が根拠付けられた |
| ⑨ | リスク評価 | リスク要因、対応策が網羅 |
| ⑩ | 実装計画 | ロードマップ、マイルストーン、KPI確定 |
| ⑪ | 財務予測 | 3～5年の財務見通しが具体化 |
| ⑫ | サマリー・結論 | 期待効果、実現性が説得力を持つ |

### 3. 表現品質の向上

- **論理性**: 各セクション間の因果関係が明確、矛盾がない
- **説得力**: 数字、根拠、エビデンスが具体的で信頼性が高い
- **読みやすさ**: 表記統一、誤字脱字なし、適切な図表使用
- **実現可能性**: 施策が具体的で、実行可能性が示されている

### 4. 建設業特有要素の完成化

- **人材戦略**: 採用、育成、定着施策の具体化
- **GX/DX戦略**: 環境・デジタル投資の具体的ロードマップ
- **地域・セグメント戦略**: 地域特性、公共/民間セグメント別施策の具体化

### 5. 文字数・フォーマット調整

- **目安**: 約10,000字程度
- **構成**: テンプレート例に従う（段落構成、見出し、図表）
- **12要素の記載**: 【12要素(1)対応】などの記載は不要のため削除する
- **最終チェック**: 表記統一、フォーマット一貫性、参考資料完全性

## 🚨 エラーハンドリング

### 「Prompt is too long」エラー対応

エラー発生時の対応手順:
1. **入力ファイルの分割読み込み**: draft_business_plan.md と validation_report.md の要約を作成してから最終版作成
2. **文書の分割作成**: 12要素を3-4要素ずつに分けて改善し、最後に統合
3. **再試行**: 最大4回まで再試行
4. **4回目の最終手段**: 3回失敗した場合、起動プロンプトを極限まで削ぎ落とし、**企業コード番号のみ**を伝達する（例: "企業コード: 12044"）。エージェントは企業コード番号から`output/[企業コード]/`配下の全成果物を自動探索し、タスクを実行する

### 前提条件未達時の対応

project_plan.md または validation_report.md が存在しない場合:
1. **エラー報告**: orchestrator に「前提条件未達: [ファイル名] なし」を報告
2. **処理中断**: 自己判断で処理を続行しない
3. **待機**: orchestrator からの再指示を待つ

### ファイル保存失敗時の対応

Write ツールでファイル保存が失敗した場合:
1. **即座に再試行**: 同一内容で再度 Write を実行
2. **3回失敗後**: エラー内容を出力に含めて報告

### 完了確認の厳格化

処理完了前の必須チェック:
```
1. Read: output/[企業コード番号]/[企業コード番号].md → 存在 & 非空確認
2. 12要素すべてが含まれていることを確認
3. 文字数が7,000字以上あることを確認
```

**ファイルが存在し、内容が空でなく、12要素が網羅されている場合のみ完了報告を行う。**
