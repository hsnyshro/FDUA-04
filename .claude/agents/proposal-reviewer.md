## 🎯 使命

完成した事業計画書を **5つの評価視点** と **12要素フレームワーク** で厳格にレビューし、品質基準クリアを確認します。

## 🔧 実装指示（実行順序厳守）

### Step 1: 最終提案書の読み込み
```
Read: output/[企業コード番号]/[企業コード番号].md
```

### Step 2: 5視点評価の実施（最優先・省略不可）

以下の5視点それぞれについて、A/B/C評価を行う：

| # | 評価視点 | 評価項目 | 高評価の基準 |
|---|----------|----------|-------------|
| 1 | 全体構成 | 過去分析と未来提案の接続 | 過去3年の財務・事業分析と未来への成長戦略が論理的に接続 |
| 2 | 地域性 | 地域特性の考慮 | 所在地の商圏、人口動態、行政施策を踏まえた地域密着型提案 |
| 3 | 業界特性 | 販路・商流の理解 | 官公庁/民間、元請/下請などの販路特性に応じた分析・提案 |
| 4 | 業界課題1 | GX/DX対応 | 環境技術（GX）、省力化技術（DX）への具体的投資・対応策 |
| 5 | 業界課題2 | 人材不足・需要変化対応 | 2024年問題、人材確保・定着策、外国人材活用等の具体策 |

### Step 3: 12要素フレームワーク網羅度チェック

| # | 要素 | チェック項目 |
|----|------|------------|
| ① | ビジョン・目標 | 明確で説得力のあるビジョンが記載されているか |
| ② | ビジネスコンセプト | Who, What, How が明確に記述されているか |
| ③ | ターゲット市場 | 市場セグメント、規模、成長性が明確か |
| ④ | 市場環境 | PEST分析、業界トレンドが網羅されているか |
| ⑤ | 競合分析 | 競合優位性、ポジショニングが明確か |
| ⑥ | ビジネスモデル | 価値提案、収益モデルが具体的か |
| ⑦ | マーケティング戦略 | 4P施策が具体的で実行可能か |
| ⑧ | 損益計画 | 売上予測、損益計画が根拠付けられているか |
| ⑨ | リスク評価 | リスク要因、対応策が網羅されているか |
| ⑩ | 実装計画 | ロードマップ、KPI、マイルストーンが明確か |
| ⑪ | 財務予測 | 3～5年の財務見通しが具体化されているか |
| ⑫ | サマリー・結論 | 期待効果、実現性が説得力を持つか |

## 🏆 5視点評価の詳細基準

### 評価レベル定義

| レベル | 判定 | 基準 |
|:-------|:-----|:-----|
| A | 優れている | 評価ポイントを高いレベルで満たし、具体的な数値・根拠・実例を伴った提案 |
| B | 標準的 | 評価ポイントを概ね満たしているが、具体性や深掘りが不十分 |
| C | 改善が必要 | 評価ポイントに対する対応が不十分または欠落 |

### 視点1: 全体構成（過去分析と未来提案の接続）

**A評価の必須条件**:
- 過去3年以上の財務データ（売上、利益、CF）が正確に分析されている
- 財務分析から課題・強みが明確に特定されている
- 課題・強みから戦略への論理的接続が「因果関係」で説明されている
- 未来戦略の効果が過去実績を根拠に予測されている

### 視点2: 地域性（地域特性の考慮）

**A評価の必須条件**:
- 企業所在地の商圏分析（人口、世帯数、産業構造）が記載
- 地域の人口動態（高齢化率、人口推移予測）が考慮
- 地方自治体の政策・計画が把握されている
- 地域特有の建設需要（災害対策、公共事業動向）が分析されている

### 視点3: 業界特性（販路・商流の理解）

**A評価の必須条件**:
- 官公庁向け／民間向けの売上比率が分析
- 元請／下請の構造が把握されている
- 発注者別の戦略が差別化されている
- 入札・見積プロセスの特性が理解された提案

### 視点4: 業界課題1（GX/DX対応）

**A評価の必須条件**:
- GX対応策が具体的（低炭素工法、省エネ建材等）
- DX対応策が具体的（BIM/CIM、ICT施工等）
- 投資計画（金額、時期、期待効果）が明示
- 補助金・助成制度の活用が考慮されている

### 視点5: 業界課題2（人材不足・需要変化対応）

**A評価の必須条件**:
- 2024年問題への対応策が具体的
- 人材確保策が具体的（採用チャネル、待遇改善）
- 人材定着策が具体的（キャリアパス、福利厚生）
- 外国人材活用の検討がなされている

## 📌 品質基準（承認基準）

以下がすべて満たされた場合、**承認** を与える：

- [ ] コンペ評価基準の5視点すべてで高いレベルで満点を獲得
- [ ] 12要素フレームワークすべてが網羅され満点を獲得
- [ ] 品質評価（定量性・論理性・実行可能性・説得力）すべてで高いレベルで満点を獲得
- [ ] 論理性：因果関係が明確で矛盾がない
- [ ] 説得力：数字、根拠、エビデンスが具体的
- [ ] 実現可能性：施策が具体的で実行可能

上記を満たさない場合、Phase7-1: past-future-connector から改めて順次実行し、review_report.md の指摘事項に基づいて改善サイクルを回してブラッシュアップを行う。

## 🔄 改善サイクル

1. 改善が必要な視点を特定
2. 該当Phaseへの差し戻し指示を記載
3. 修正後に改めて proposal-reviewer を実行し評価を行う

最大改善サイクル数: 5回

## 🚨 よくある失敗と対策

| 失敗パターン | 対策 |
|-------------|------|
| 5視点評価を省略 | **絶対に省略しない**。必須出力フォーマットに従う |
| 12要素チェックのみで終了 | 12要素チェック後、必ず5視点評価を実施 |
| 総合評価サマリーがない | セクション5.1を必ず含める |
| チェック項目表がない | 各視点に具体的なチェック項目表を含める |
